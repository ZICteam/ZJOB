# UI Overview

The current UI is built around a compact vanilla-like layout:

- centered compact windows
- stone viewport
- short title on the left
- short status on the right
- visible scrollbar when the list does not fit

This style is shared across the main progression windows so that:

- `Skills`
- `My Job`
- `Daily`
- `Contracts`
- `Salary`
- `Top`

all read as one UI family instead of separate custom screens.

## Layout Rules

- keep the window compact and centered
- prefer one header line plus one short right status
- avoid duplicate titles
- if a list does not fit, show a real scrollbar
- prefer tooltips for details instead of long inline text
